My Learning Journey

This page collects everything I learned and practiced while building skills in networking, Linux, bug bounty, and AI-assisted learning. It's long — dive into sections that interest you.

Networking Fundamentals

Networking is the backbone of everything online. Key topics I learned and practiced:

  • HTTP & HTTPS — requests, responses, status codes, headers, cookies, sessions, and how HTTPS secures transport via TLS.
  • DNS — resolving domain names, record types (A, AAAA, CNAME, MX, TXT), and DNS enumeration techniques.
  • TCP/UDP — reliable vs connectionless transports, when services use each, and how that affects testing.
  • Ports & Scanning — Nmap usage patterns, service fingerprinting and banner grabbing to identify running services.

Linux & System Administration

Linux is my lab environment. Essentials I practiced:

  • Navigation and file tooling: ls, cd, find, cat, grep
  • Permissions & special bits: chmod, chown, SetUID, how SetUID binaries can be abused for privilege escalation when misconfigured
  • Process and service control: systemctl, ps, top, and securing services with firewalls.
  • Using VMs and containers to safely test exploits and tools.

Practical Tools

Nmap

Port scanning, NSE scripts, OS detection and how to interpret scan results for recon and prioritizing targets.

Netcat

Socket interactions, reverse shells, and simple data transfer — the swiss-army knife for quick testing.

Burp Suite & Browser DevTools

Intercepting HTTP traffic, modifying requests, learning to read JSON and form data for vulnerability discovery.

Bug Bounty & Web Security

What I focused on learning and practicing:

  • Understanding stateful vs stateless systems, session & cookie flows, CSRF, XSS, SQLi basics.
  • Practicing labs in PortSwigger Web Security Academy and writing proof-of-concept exploit steps clearly.
  • Learning how to use AI to summarize request flows, suggest test cases, and draft bug reports — while ensuring manual verification.

AI in the Loop

How I use AI (LLMs) to accelerate learning:

  • Breaking down complex topics into simpler steps and checklists.
  • Generating scaffolded practice exercises and creating payload templates for testing APIs.
  • Drafting clear writeups for bug reports based on manual observations.

Practical Next Steps (Roadmap)

  • Learn Burp Suite advanced flows — Repeater, Intruder, Collaborator usage.
  • Build automation scripts to parse Nmap output and highlight likely attack paths.
  • Create a reproducible lab environment for Web & API testing and keep documenting progress on this site.
